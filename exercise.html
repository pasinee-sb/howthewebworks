HTTP is Hyper Text Transfer Protocol is governs how clients get data from, or send data to, a server. URL is Uniform Resource Locator, an address for internet resource. Short for Domain Name System, a system that takes human-readable URLs and converts them into IP addresses. The query string allows you to pass key-value pairs into the URL, in the format ?key1=value1&key2=value2... GET - get data from the server POST - send data to the server An HTTP request is a request from a client to a server which follows the HTTP protocol An HTTP response is a response from a server to a client which follows the HTTP protocol (eg sending back HTML/CSS/JS/etc) Headers provide additional information about the request or the response. ex. Request headers: Host, User-Agent, Accept, Cookie, Cache-Control Response headers: Content-Type, Last-Modified, Set-Cookie, Cache-Control Browser resolves the name into an IP address using DNS and makes a request to that IP address, including headers (info about browser, any previous cookies, and other things) The server sends a response (typically, HTML, with a status code (200 if it was sucessful) The browser makes a DOM from that HTML, and finds any other resources needed (images, CSS, JavaScript, etc) then makes separate HTTP requests for those resources and receives response from the server for each $curl htcurl -H "Accept: application/json" "https://icanhazdadjoke.com/search?term=pirate" $dig http://www.icanhazdadjoke.com Part Three: Explore Dev Tools Build a very simple HTML form that uses the GET method (it can use the same page URL for the action) when the form is submitted.
